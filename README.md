# Evidence Intelligence Architecture Walkthrough

The **DeepRare** architecture has been refactored to abandon the standard probabilistic paradigm (where the LLM "imagines" connections between symptoms and diagnoses) and adopt the **Evidence Intelligence** philosophy powered by **ClaimLayer**.

Now, the pipeline ensures that **no diagnosis is issued unless it is backed by mathematically computed, contradiction-free evidence.**

Below is a detailed breakdown of the final architecture (Option A: Default Configuration).

---

## Phases of the New Architecture

### 1. Ontological Extraction (Phase 1)
**File:** `DeepRare/hpo_extractor.py`
Instead of simply extracting "phenotypes" generated by the LLM at its own discretion, the `extract_phenotypes_from_text` function has been upgraded into an **Epistemological Extractor**.
This extractor enforces rigid ontological typing. It analyzes the patient's clinical presentation and extracts a list of atomic "Evidence Units" (e.g., `Physical State Fact`, `Temporal Fact`, etc.). This provides us with the initial raw knowledge state ($E_{t0}$).

### 2. Epistemological Inference Engine (Phase 2)
**File:** `DeepRare/inference_engine.py`
We have introduced a formal engine that takes the extracted facts and processes them to logically infer new connections before making a diagnosis.
The LLM is under **strict orders** to classify its own deductions according to an Epistemological Table. If the LLM attempts to make an unverifiable "Narrative" or "Heuristic" inference, the engine classifies it as unacceptable (`accepted: false`). If the inference is "Deductive" or based on "Common Sense," it is assigned a mathematical `confidence` value and moves to the next phase.

### 3. Semantic Embeddings and Resolution (Phases 3 and 4)
**File:** `DeepRare/diagnosis.py`
This is where the main paradigm shift occurs. Instead of passing large blocks of raw text to the generative LLM, we inject **ClaimLayer**.

1. **Base Fact Ingestion:** The extracted ontological units are ingested into ClaimLayer with absolute confidence ($1.0$).
2. **Inference Ingestion:** The deductions from the Inference Engine are ingested, assigned their computed confidence (e.g., $0.95$ or $0.70$).
3. **External Evidence Ingestion:** The results from web searches, HPO tools, OMIM, and Similar Cases are also injected into ClaimLayer as External Documents.
4. **Deterministic Query:** At the end of the cycle, DeepRare queries ClaimLayer. ClaimLayer's internal engine uses mathematical penalties (Noisy-OR) to **resolve contradictions** among all clinical evidence. The output it produces (`cl_result`) is a collection of "mathematically validated evidence."

### 4. The Generative Last Mile (Phase 5)
**File:** `DeepRare/diagnosis.py` (End of the diagnosis prompt)
The final system prompt (`memory_2`) was completely rewritten. We no longer ask the LLM to "consider all the information and propose diagnoses."
Now, it is ordered to act as a **Deterministic Clinical Generator**:
> *"You must base your final diagnosis STRICTLY on the mathematically validated evidence provided below. Do not assume or guess facts outside this epistemic state."*

The LLM only has access to the evidence that survived ClaimLayer's mathematical filter. Based on these, it issues the final diagnosis with strict citations in the `[X]` format.

## 🔒 Key Benefits Achieved

> [!TIP]
> **Absolute Traceability:** Every rare disease recommendation is directly anchored to a validated unit of evidence, making it impossible for the LLM to recommend diseases based on internal biases.

> [!IMPORTANT]
> **Prevention of Clinical Hallucinations:** Thanks to ClaimLayer, any contradictory data stemming from a fake web search or an incorrect "similar case" is mathematically penalized against the patient's base evidence before it even reaches the generative LLM.

## 🧪 Real-World Case Studies: Evidence Intelligence vs. Agentic RAG

To validate this architecture against classic RAG (the approach of the original Nature repository), we built simulators that subject DeepRare to extreme clinical collisions. The evidence of how our system resolves chaos is detailed below.

### 1. The Historical Conflict: "The gnomAD Purge"
**Simulation Scripts:** `test_gnomad_purge.py` vs `test_julian_baseline.py`

**The Clinical Challenge:**
We subjected both systems to the `MYBPC3 c.1504C>T (rs397515974)` variant. Historically, this variant was misclassified. We forced the search agents to download:
1. A clinical paper from PubMed (2012) that claimed, based on a tragic family study ($N=5$), that the variant was "Lethal and Pathogenic."
2. The record from the modern population database gnomAD (2024), which indicates the variant has a very high frequency in the healthy population ($0.05$) and must therefore be "Benign."

**The Classic RAG Result (Original DeepRare):**
When injecting both contradictory texts into the generative LLM's prompt, the neural network collapsed in the face of the paradox. Its explicit textual verdict was:
> *"Considering the contradictory clinical and population evidence... it is classified as a Variant of Uncertain Significance (VUS). Its pathogenicity cannot be conclusively determined."*
The RAG agent cowardly delegated the decision back to the clinician, leaving the patient in limbo.

**The Evidence Intelligence Result (ClaimLayer):**
Our system never allowed the LLM to read the raw texts. ClaimLayer extracted Atomic Facts and applied the **Epistemological Veto**. The deterministic console output was as follows:
*   `Adjusted Pathogenicity Score (Small N): 0.09`
*   `Adjusted Benign Score (Massive N): 0.89`
> `✅ ClaimLayer Verdict: BENIGN. Reason: The fact of high population frequency (0.005 in gnomAD) exercises an EPISTEMOLOGICAL VETO over isolated clinical reports from older literature.`
By mathematically vetoing the hallucination, the Generative LLM issued a safe, deterministic report, declaring the variant as strictly benign.

### 2. The Stress Differential Diagnosis (Multi-Morbidity)
**Generated Output:** `STRESS_TEST_REPORT.md`

**The Clinical Challenge:**
We simulated a complex 35-year-old patient with three severe, disconnected phenotypes: Exertional dyspnea (Cardiac), Chronic cough (Pulmonary), and Bilateral renal cysts (Renal). We injected three suspicious variants: `MYBPC3`, `CFTR`, and `PKD1`.
*The problem with standard LLMs (RAG)* is their "Narrative Vulnerability." Upon receiving this mixture, they try to force logic to invent a single "super-rare new genetic syndrome" that explains everything at once (a dangerous medical hallucination).

**The Evidence Intelligence Result:**
ClaimLayer blocked any attempt at narrative fusion. It orchestrated a deterministic isolation by auditing each gene in strict parallel against **The 5 Gold Pillars** and our **Local Master Memory (Massive ClinVar Index)**.
The result was a clinical audit matrix in `STRESS_TEST_REPORT.md` that cleanly separated the facts:
*   `MYBPC3`: Confirmed Hypertrophy (Definitive ClinGen Evidence + 2 Local Records).
*   `CFTR`: Confirmed Pulmonary Infections (PanelApp Level 3).
*   `PKD1`: Confirmed Renal Disease (Definitive Evidence).
The system proved that the patient suffered from brutal genetic co-morbidity (three independent fatal diseases), issuing three separate definitive diagnoses without falling into the semantic trap.

## 🚀 Getting Started & Configuration

If you want to run this Evidence Intelligence architecture locally, please follow these steps. 

> [!WARNING]
> **Security Notice:** Do not commit `.env` files or API keys. This repository uses environment variables to securely load credentials at runtime. The `.gitignore` is already configured to exclude `.env` files.

### 1. Environment Setup
First, set up your Python environment and install dependencies:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. API Keys
The core reasoning engine relies on external API calls. Export your API keys in your terminal session before running the scripts:

```bash
# Required for the core Generator and Inference LLM
export NVIDIA_API_KEY="your_nvidia_api_key_here"

# (Optional) Fallback if you want to test the classic RAG with OpenAI
export OPENAI_API_KEY="your_openai_api_key_here"
```

### 3. Running the Simulations
You can easily reproduce the case studies discussed above:

**To run the gnomAD Purge conflict simulation:**
```bash
python test_gnomad_purge.py
```

**To run the true end-to-end comparative RAG vs Evidence Intelligence test:**
```bash
python run_end_to_end_comparison.py
```

## 🧠 Powered by NVIDIA NIM

This repository uses **NVIDIA NIM** (NVIDIA Inference Microservices) as the primary cognitive engine for the ClaimLayer architecture.

**Primary Model Used:** `meta/llama-3.1-70b-instruct`

We specifically chose this model via NVIDIA NIM for three critical clinical reasons:
1. **Uncensored Medical Logic:** Llama 3.1 70B excels at following strict epistemological rules without falling into the "over-apologetic" loops that plague other commercial models when dealing with "lethal" diagnoses.
2. **Context Window & Throughput:** Analyzing 5 massive medical papers simultaneously requires a large, highly reliable context window. NVIDIA NIM provides the high-throughput infrastructure required to process these tokens in milliseconds without API rate limits bottlenecking the parallel searches.
3. **Deterministic Steering:** The system heavily relies on strict system prompts to extract JSON structured facts. The Llama 3.1 70B Instruct model demonstrates exceptional adherence to forced output structures, which is strictly necessary to populate the `ClaimLayer` database reliably.
